## Зачем Jarvis

Когда гоняешь несколько кодинг-агентов разом, **узким местом становишься ты сам**. Сессии раскиданы по терминалам, вкладкам и Spaces — и с одного взгляда не видно, какой агент **встал на запросе разрешения**, какой **закончил полчаса назад и простаивает**, пока ты думаешь, что он работает, а какой **упёрся в лимит**. Родные уведомления агентов — по одной сессии и привязаны к терминалу (в расширении для VS Code их вообще нет), единой картины по всем сессиям нет. А уснувший мак **молча морозит** процессы агентов и рвёт API-запросы, убивая долгие ночные прогоны — и на Apple Silicon закрытая крышка форсит сон, который обычный `caffeinate` не обходит.

**Jarvis — это та самая недостающая единая панель.** Он живёт в меню-баре, видит каждую интерактивную сессию Claude Code и Codex CLI на машине и следит, чтобы ни один агент не ждал тебя молча:

- ты **видишь** общую картину постоянно (живые счётчики в меню-баре, панель ⌘J в духе Raycast);
- ты **слышишь** её — тосты поверх фуллскрина и опциональный локальный голос, озвучивающий события сессий;
- ты **действуешь**, не разыскивая нужный терминал — отвечаешь прямо в сессию, переключаешь модель и effort, а мак не засыпает всю ночь.

## Что умеет

- **🖥 Мульти-сессионный монитор** — все сессии Claude Code и Codex во всех терминалах, живые счётчики **⏸ ждут · ⚙ работают** в меню-баре.
- **🔔 Тосты поверх фуллскрина** — «нужно разрешение» видно даже в полноэкранном приложении; содержимое настраивается (ветка, модель, effort, токены, время).
- **🎛 Панель ⌘J, always-on-top** — открыл, глянул, сделал, закрыл (как Raycast); фокус не ворует.
- **↩️ Ответ прямо в сессию** — печатаешь в сессию через tmux, даже если окно свёрнуто или на другом Space; есть палитра `/`-команд.
- **⚙️ Пульт** — модель (Opus / Sonnet / Haiku) и effort переключаются из панели; на вопросы агента с вариантами — нативные пикеры (включая мультивыбор).
- **📊 Расход, стоимость и лимиты** — учёт токенов и стоимости по моделям и проектам; упёрся в лимит — Jarvis покажет, когда сброс, и может сам продолжить сессию.
- **🗣 Jarvis говорит** — локальный TTS озвучивает, что сессия сделала или чего ждёт (пока по-русски).
- **🎤 Голосовой ассистент «Hey Jarvis»** *(экспериментально)* — скажи wake-word и говори с сессиями: ответ голосом в нужную сессию, «что сделал агент?», музыка/громкость, общий вопрос.
- **🎙 Диктовка** — push-to-talk (F8): речь распознаётся локально (Whisper / Qwen3) и вставляется в активную сессию; полная история диктовок с пере-распознаванием.
- **☕ Не давать маку уснуть** — анти-сон (аналог `caffeinate`) и страхуемый режим закрытой крышки для ночных прогонов.
- **✅ Доска задач (read-only)** — живой прогресс `TodoWrite` (готово / в работе / в очереди) по сессии.
- **📦 Менеджер моделей** — скачивание, удаление и горячая смена локальных TTS/STT/wake-word-моделей из настроек; онбординг при первом запуске.
- **🔒 Событийно и приватно** — на родных хуках агентов (без чтения экрана), всё локально, без телеметрии, сносится одной командой.

> **Граница:** Jarvis — **монитор и пульт, не оркестратор.** Он не плодит агентов и не владеет планом; он дополняет оркестраторы (Claude Squad, Conductor, Crystal, нативные Agent Teams) и те сессии, что ты уже запустил.

## Кому это нужно

Jarvis — инструмент для power-пользователей. Он окупается, когда ты перешёл порог боли в **3+ параллельные агентные сессии**:

- **Соло-разработчик с флотом агентов** — несколько сессий Claude Code / Codex по разным проектам, где ручная проверка каждого терминала съедает то самое время, которое агенты должны были сэкономить.
- **Ночные и долгие прогоны без присмотра** — большие рефакторинги, циклы «тест-фикс», массовые миграции на MacBook: Jarvis не даёт маку уснуть (даже с закрытой крышкой) и сам продолжает сессии после сброса лимита.
- **Те, кто любит работать не отвлекаясь** — ты пишешь код или читаешь, пока агенты молотят; Jarvis дёргает тебя только когда одному из них реально нужно решение — тостом или голосом.
- **Небольшие команды с агентным воркфлоу** — Jarvis полностью локален (ни сервера, ни аккаунтов): каждый разработчик просто ставит его себе и получает одинаковую картину.

Скорее всего, Jarvis **не** нужен, если у тебя одна сессия в одном терминале (родных уведомлений может хватать) — или если ты ищешь то, что будет *плодить и планировать* агентов: для этого поставь рядом оркестратор.

## Установка

### Из релиза (рекомендуется)

1. Скачай `Jarvis_x.y.z_aarch64.dmg` со страницы [releases](https://github.com/Sergey-Chernyshev/jarvis/releases).
2. Открой DMG, перетащи **Jarvis** в **Applications**, запусти.
3. Первый запуск: сборка подписана **ad-hoc** (без Apple Developer ID, без нотаризации), поэтому macOS предупредит о неизвестном разработчике. Правый клик по приложению → **Открыть** → **Открыть**. Если macOS пишет, что приложение «повреждено», сними карантин:

   ```bash
   xattr -dr com.apple.quarantine /Applications/Jarvis.app
   ```

4. При первом запуске Jarvis предложит настроить интеграцию с Claude Code / Codex и скачать голосовые модели — нажми **«Настроить»** (прогресс по шагам прямо в окне).

Обновления приложение проверяет само (встроенный updater). Переустановить интеграцию можно в любой момент: меню-бар → «Переустановить интеграцию…».

**Требования:**

- мак на **Apple Silicon** (M1 и новее), macOS 11+ — готовый DMG собирается только под aarch64; на Intel можно собрать из исходников (ниже);
- **tmux** для ответа-в-сессию и пульта: `brew install tmux`;
- **Claude Code** (CLI) и/или **Codex** (CLI) — агенты, за которыми Jarvis следит.

<details>
<summary>Сборка из исходников / для разработчиков</summary>

Нужны: Rust stable ([rustup](https://rustup.rs/); минимальная версия — поле `rust-version` в [`src-tauri/Cargo.toml`](src-tauri/Cargo.toml)), Node.js 20+, CMake (`brew install cmake`), tmux (опционально).

```bash
npm ci
npm run setup     # вшить хуки в ~/.claude/settings.json (с бэкапом, идемпотентно)
npm start         # собрать и поднять демон + меню-бар (◇ справа сверху) на dev-профиле (~/.jarvis-dev)
```

(npm здесь — только привычный раннер: внутри это `cargo build --release` плюс ad-hoc `codesign`, нужный для доступа к микрофону.)

Дальше **перезапусти активные сессии** `claude` — хуки снимаются снапшотом на старте сессии. Снести всё: `npm run teardown`. Посмотреть состояние каждого компонента: `npm run status`. Локально собрать DMG (без подписи, host-арка): `npm run bundle`. Запустить прод-профиль: `npm run start:prod`.

**Пощупать без Claude Code** (демон должен быть запущен) — вручную, как это делает настоящий хук:

```bash
echo '{"session_id":"t1","cwd":"'$PWD'"}' | ~/.jarvis/bin/jarvis-hook claude session-start
echo '{"session_id":"t1","cwd":"'$PWD'","prompt":"add tests"}' | ~/.jarvis/bin/jarvis-hook claude prompt
echo '{"session_id":"t1","cwd":"'$PWD'","message":"Claude needs your permission to use Bash"}' | ~/.jarvis/bin/jarvis-hook claude notification
echo '{"session_id":"t1","cwd":"'$PWD'"}' | ~/.jarvis/bin/jarvis-hook claude stop
```

Проверка, что демон жив: `curl -s --unix-socket ~/.jarvis/run.sock http://jarvis/`

</details>

## Возможности

### 🖥 Мониторинг сессий

Реестр всех интерактивных сессий Claude Code / Codex: статус (idle / работает / ждёт / закончила / упёрлась в лимит), проект, ветка, модель, активность. Счётчик в меню-баре (**◇ ⏸N ⚙M**) — сводка по всем терминалам; панель ⌘J — детальный вид.

<details>
<summary>Как это работает</summary>

Никакого парсинга экрана — только структурированные события от самих агентов через их родные хуки: `SessionStart` → idle, `UserPromptSubmit` → working, `Notification` → waiting (+уведомление), `Stop` → done (+уведомление), `SessionEnd` → сессия исчезает. Реестр-редьюсер живёт в Rust-демоне; состояние переживает перезапуск демона (`~/.jarvis/state.json`), живые tmux-сессии переусыновляются. Экран паны читается только детектором интерактивных промптов и пультом (регуляркой, не скриншотами).

</details>

### 🔔 Уведомления и 🎛 панель ⌘J

Собственные тосты-уведомления рендерятся **поверх фуллскрина**; панель (`alwaysOnTop`) не ворует фокус. Глобальный хоткей **⌘J** открывает панель по центру экрана — Esc, клик мимо или повторный ⌘J закрывают.

<details>
<summary>Подробности</summary>

Клик по ◇ — панель. Правый клик — меню (тест уведомления, автозапуск, выход). Хоткей, позиция панели, тумблеры уведомлений и автозапуск — в настройках (⚙ в шапке панели), хранятся в `~/.jarvis/settings.json`. Содержимое уведомлений собирается из сегментов (ветка · модель · effort · токены · время) с живым превью; отдельный тумблер озвучивает уведомления только при подключённой Bluetooth-гарнитуре. Учти: глобальный ⌘J перехватывает эту клавишу во всех приложениях (Chrome «загрузки», VS Code «панель») — поменяй сочетание в настройках, если мешает.

</details>

### ↩️ Ответ в сессию и ⚙️ пульт

В чате сессии поле **«Ответить»** вставляет текст прямо в терминал сессии — даже если окно свёрнуто или на другом Space. Под полем — **пульт**: сегменты **Модель** (Opus / Sonnet / Haiku) и **Effort** (auto / low / med / high / xhigh) шлют слэш-команду (`/model sonnet`, `/effort high`) в живую сессию. Когда агент задаёт вопрос с вариантами, панель рисует нативные пикеры (включая мультивыбор) — не нужно печатать номера в терминал.

<details>
<summary>Как это работает</summary>

`npm run setup` ставит PATH-шим `~/.jarvis/shims/claude` (паттерн pyenv, managed-блок в `~/.zshrc` между маркерами jarvis). После `exec zsh` каждый интерактивный запуск `claude` прозрачно оборачивается в tmux на **отдельном сервере** (`-L jarvis`, конфиг `~/.jarvis/tmux.conf`): без статус-бара, мышь скроллит, личный tmux не затрагивается. В iTerm2 — control mode (`-CC`), родные табы. Headless-запуски (`-p`, пайпы, `$TMUX`) не оборачиваются. Вставка: `set-buffer → paste-buffer -p → send-keys Enter` (многострочные промпты доходят одним куском). Палитра команд (`/` в поле ответа) даёт доступ к остальным слэш-командам.

- Сессия вне tmux помечена «вне tmux» — управлять ей нельзя; панель подскажет, как завести её в tmux (`claude --resume <session_id>`).
- Закрыл окно терминала — агент жив: tmux-сессия уходит в detach, Jarvis продолжает следить. Переподключиться: `tmux -L jarvis attach -t <имя>`.
- **Модель** видна бесплатно (пишется в transcript каждым ходом ассистента). **Effort** снаружи не читается — панель ведёт оптимистичное состояние, подсвечивая то, что выставила сама.

</details>

### 🤝 Claude Code и Codex — бок о бок

Если при установке найден `codex`, Jarvis подключает и его: хуки в `~/.codex/hooks.json` и шим `codex` на том же tmux-механизме. Интерактивные сессии Codex появляются в панели с бейджем `codex` и получают всё то же: статус, тосты, голос, чат, ответ-в-сессию и команду продолжения (`codex resume <session_id>`).

<details>
<summary>Особенности Codex</summary>

Модель и reasoning меняются через родной `/model`-пикер Codex (отдельного `/effort` у него нет). **Headless `codex exec` хуки не шлёт** — такие запуски не мониторятся (по дизайну, как и `claude -p`). На свежей машине хуки Codex требуют доверия (`~/.codex/config.toml [hooks.state]`); шим Codex добавляет `--dangerously-bypass-hook-trust`, если установленный `codex` поддерживает флаг, — это **снимает проверку доверия для всех хуков Codex** при интерактивном запуске, имей этот компромисс в виду. Цифры расхода/стоимости для Codex — оценочные.

</details>

### 📊 Расход, стоимость и лимиты

Jarvis считает токены и оценочную стоимость по сессиям, моделям и проектам и понимает скользящее окно лимита. Когда сессия упирается в лимит, панель и голосовая реплика говорят, когда сброс, — а со включённым авто-резюме Jarvis сам продолжит сессию, как только окно откроется.

### 🗣 Голос: Jarvis говорит (TTS)

После события сессии Jarvis коротко проговаривает, что произошло, локальным TTS — пока по-русски. Текст собирается шаблоном из структурных сигналов; числа разворачиваются в слова с русским согласованием.

- ход закончен → «Пиксела: четыре из шести задач, сейчас docker-compose» (есть доска) · «Рекрю готов, изменено три файла» (есть diff);
- ждёт тебя → **приоритетно**: «Пиксела ждёт — нужно разрешение на Bash»; упёрлась в лимит → «Пиксела упёрлась в лимит, сброс через два часа».

<details>
<summary>Движок, конфиг, граница</summary>

Движок — **Silero** (локальный Python-сайдкар, FastAPI, модель в памяти; слушает только `127.0.0.1`), которым демон рулит сам: поднимает по требованию, перезапускает при падении, гасит на простое и на выходе. Спикеры: `aidar` · `baya` · `kseniya` · `xenia` · `eugene`. Реплики не накладываются: очередь сериализована, «ждёт» идёт вперёд «готово», затор `Stop`-реплик коалесцируется. Конфиг — в `~/.jarvis/settings.json`:

```json
"voice": { "engine": "silero", "mute": false,
           "events": { "stop": true, "notification": true, "stopFailure": true } }
```

Меню-бар: тумблер **«Без звука»** и **«Тест голоса»**. Если движок недоступен — демон работает как раньше, агенты не затронуты, причина в логе.

⚠️ **Лицензия голоса:** дефолтный русский голос Silero `v4_ru` — **некоммерческий** (CC BY-NC-SA 4.0). Для коммерческого использования — см. раздел [«Лицензия»](#лицензия).

</details>

### 🎤 «Hey Jarvis» — голосовой ассистент *(экспериментально)*

Скажи wake-word и говори — без клавиатуры и переключения окон:

- **ответ голосом в сессию** — Jarvis по содержанию понимает, к какой сессии ты обращаешься (а при сомнении показывает пикер), подставляет текст и отправляет;
- **«что там у агента?»** — «что сделала сессия тикетинга?» — и получаешь голосовую сводку;
- **быстрые OS-команды** — плей/пауза/следующий трек, громкость, открыть приложение;
- **общие вопросы** — уходят отдельному локальному ассистент-агенту, который умеет искать в вебе и отвечает голосом.

Диалог полудуплексный: пока Jarvis говорит — он не слушает, а пока говоришь ты — уведомления ждут. Детектор wake-word (openWakeWord) включён в релизные сборки, но **по умолчанию выключен** — включается в панели; модель детектора скачивается по запросу (некоммерческая лицензия — см. [«Лицензия»](#лицензия)). Верификации говорящего пока нет — wake-word может сказать кто угодно в комнате.

### 🎙 Диктовка (push-to-talk STT)

Зажми **F8**, говори, отпусти — речь распознаётся локально и вставляется в активную сессию. Движок по умолчанию — Qwen3-ASR через MLX-сайдкар; Whisper (whisper.cpp, Metal) встроен как альтернатива. Смешанная русско-английская речь — включая термины и код — распознаётся нормально.

<details>
<summary>История диктовок и анти-галлюцинации</summary>

Панель хранит полную историю диктовок: поиск, группировка по дням, статистика, улучшение расшифровки, экспорт, удаление — а последние диктовки хранят сжатое аудио, так что их можно **пере-распознать** другим движком. Против галлюцинаций STT на тишине/шуме — VAD-гейт (Silero, альфа-тумблер, по умолчанию выключен), блоклист фраз и подобранные параметры декодирования.

</details>

### ☕ Не спать · ⌒ Крышка (power-плагины)

Уснувший мак — замороженные процессы агентов и оборванные API-запросы. Два подключаемых плагина (тумблеры в настройках), UX списан с Raycast Coffee и Amphetamine:

- **☕ Не спать** — вето на idle-сон через power assertion (IOPMAssertion, как у `caffeinate`): бессрочно, на время (15м…8ч), пока жив процесс, или авто — пока агенты работают.
- **⌒ Крышка** — closed-display mode (`pmset disablesleep`): мак работает с закрытой крышкой. На Apple Silicon это то, что обычный `caffeinate` не даёт.

<details>
<summary>Безопасность и страховка</summary>

Assertion живёт в процессе демона: краш = автоснятие, «застрявший» запрет сна невозможен. Проверка: `pmset -g assertions | grep -i jarvis`.

«Крышка» — это root и термо-риски, поэтому плагин **детектит и подсказывает, а не молча sudo**: после прерванного сна предлагает включить closed-display; с внешним дисплеем советует родной clamshell (root не нужен). Ручной тумблер — честный admin-диалог; для тихого режима — «Настроить тихий режим» (`/etc/sudoers.d/jarvis-pmset`, ровно две команды через `visudo -c`). Fail-safe: маркер `~/.jarvis/clamshell.json`, восстановление сна на старте/выходе демона, батарейный сторож (≤15% → вернуть сон). MacBook Air без вентилятора под крышкой троттлит — плагин предупреждает.

</details>

### ✅ Доска задач (наблюдатель)

Сессии с многошаговым планом (`TodoWrite`) показывают кольцо «N/M задач» → слайд-овер «Задачи»: агрегат, прогресс-бар, список со статусами, где удаётся — модель и время сабагента. Доска живая: следующий `TodoWrite` перерисовывает её сам.

<details>
<summary>Почему read-only</summary>

Источник истины по задачам — **оркестратор внутри сессии**. У списка `TodoWrite` нет внешнего API «отметь сделанным» — мутировать извне нельзя. Jarvis доску **только читает** (`PostToolUse`, last-write-wins) и **отображает**. Действие с задачи (перейти / пропустить / перезапустить) — это **инструкция оркестратору**: префилит поле ответа редактируемым текстом, который ты правишь и отправляешь сам. Доска меняется только когда прилетит следующий реальный `TodoWrite` — так видно, что агент реально делает, а не что мы попросили.

</details>

### 📦 Менеджер моделей

Все локальные модели — голоса TTS, движки STT, детектор wake-word — управляются из одного раздела «Модели» в настройках: статус, размер на диске, скачивание, удаление и горячая смена активного STT-движка без перезапуска демона. Онбординг первого запуска предлагает чек-лист моделей с единым прогрессом загрузки. В приложение ничего не вшито; веса скачиваются из первоисточников в `~/.jarvis/`.

| Ответ в сессию | ✅ (tmux) | ❌ | ✅ | — |
| Пульт модель/effort | ✅ | ❌ | частично | вручную |
| Голос (TTS + ассистент) | ✅ | ❌ | ❌ | ❌ |

Ближайший по духу — мониторы меню-бара; их козырь — нулевая настройка. Jarvis отвечает точностью (хуки, а не догадки), ответом-в-сессию с пультом, голосом и управлением питанием.

## Как это работает

```
claude / codex (любой терминал)
  └─ родные хуки (~/.claude/settings.json · ~/.codex/hooks.json)  ← вшивает setup
       └─ ~/.jarvis/bin/jarvis-hook        ← fail-silent шим, curl за 0.3с
            └─ unix-сокет ~/.jarvis/run.sock
                 └─ Rust-демон (Tauri) = реестр сессий + эффекты
                      ├─ тосты поверх фуллскрина · панель ⌘J (always-on-top)
                      ├─ tmux-ответ и пульт · TTS/STT-сайдкары · power-плагины
                      └─ счётчик в меню-баре: ⏸ ждут · ⚙ работают
```

Фронтенд (панель `ui/index.html` + `ui/renderer.js`, тосты `ui/toast.*`) работает в системном WKWebView; контракт `window.jarvis` реализует тонкий адаптер `ui/bridge.js` поверх Tauri IPC. Main-процесс на Rust (`src-tauri/src/`): unix-сокет на axum, реестр-редьюсер с эффектами, tmux, транскрипты, usage/история, голос, STT, wake-word, плагины питания. Состояние, настройки и статистика лежат в `~/.jarvis/` в стабильных форматах.

**Принципы:** без чтения экрана (только структурные события) · хуки fail-silent (никогда не ломают агентов) · всё локально, без телеметрии · снести всё одной командой (`npm run teardown`). История разработки по инкрементам — спеки, планы, мокапы — в [`docs/superpowers/`](docs/superpowers/).

## Советы и рекомендуемая настройка

- **Поставь tmux** (`brew install tmux`) — без него Jarvis мониторит и уведомляет, но не может отвечать в сессии и рулить пультом. В iTerm2 сессии получают родные табы (tmux control mode).
- **Работай в паре с оркестратором.** Claude Squad, Conductor, Crystal, нативные Agent Teams — они спавнят и планируют; Jarvis показывает всё, что они (и ты) запустили интерактивно, в одном месте.
- **Ночные прогоны:** включи **«Не спать» → авто** (не спит, пока агенты работают), доверь крышку clamshell-страховке и включи авто-резюме — сброс лимита не оставит прогон стоять.
- **Голос в офисе:** тумблер «озвучивать только в Bluetooth-гарнитуру» не даст сводкам звучать из колонок.
- **⌘J конфликтует** с «Загрузками» Chrome и панелью VS Code — перебинди в настройках, если пользуешься.
- **Коммерческое использование:** дефолтные модели голоса и wake-word некоммерческие; переключись на чистый набор (Silero `v5_cis_base`, Whisper/Qwen3, свой wake-word или без него) — см. [THIRD-PARTY-NOTICES.md](THIRD-PARTY-NOTICES.md).
- Сессия, запущенная мимо шима, помечается «вне tmux» — перезапусти её через `claude --resume <session_id>`, чтобы управлять.

## Планы

Jarvis — соло-проект в стадии pre-1.0; это направление, а не обещание. Лучший способ повлиять — [issue](https://github.com/Sergey-Chernyshev/jarvis/issues).

**Голосовой ассистент → mission control без рук**

- Создание *новой* сессии голосом («Jarvis, запусти агента в ~/myproject») — сейчас голос только маршрутизирует в существующие.
- Акустический barge-in: перебить Jarvis на полуслове так, чтобы микрофон не ловил его собственную речь (эхоподавление через macOS `VoiceProcessingIO`).
- Стриминговая пословная диктовка (сейчас распознавание — по отпусканию клавиши).
- Постоянная память ассистента между разговорами; отдельная вкладка истории разговоров.
- Верификация говорящего для wake-word (сейчас триггернуть может кто угодно в комнате).
- Полноценный голосовой HUD.

**Агентная платформа**

- Дошипать UI агент-чата поверх capability-платформы — гейт безопасности и MCP-мост `jarvis-mcp` уже на месте.
- Система плагинов (манифесты, песочница, подпись) и следующие слои безопасности (span-провенанс, контроль исходящего трафика).
- Отдача возможностей Jarvis наружу как MCP-сервера для внешних клиентов.

**Движки и модели**

- Полностью нативный TTS — без Python-сайдкара (Piper через `ort` или Silero через `ort` с нативной расстановкой русских ударений).
- Квантованные STT-модели (4-bit Qwen3) после проверки точности; шумодав продакшн-уровня (текущий тумблер — альфа).

**Паритет Codex** — учёт лимитов с авто-резюме и точные (а не оценочные) цифры расхода для сессий Codex.

**Дистрибуция и полировка**

- Настоящая подпись Developer ID и нотаризация (сейчас сборки ad-hoc).
- Демо-GIF и скриншоты; готовый Intel-DMG, если будет спрос.
- `1.0`: заморозка контракта хуков и форматов на диске.

**Исследуем** — английский UI и английские голосовые сводки (интерфейс и голос пока русскоязычные).

**Сознательно вне scope:** запуск/оркестрация агентов (намеренно — см. границу выше), Windows/Linux (проект глубоко macOS-нативный: IOPMAssertion, `pmset`, WKWebView, MLX) и Mac App Store (его песочница несовместима с хуками и tmux).

## Статус и ограничения

Pre-1.0 MVP, разрабатывается открыто. Сознательные границы и known-issues:

- **macOS-only**; поддержаны **Claude Code (CLI)** и **Codex (CLI)** — только **интерактивные** сессии (headless `claude -p` / `codex exec` хуки не шлют — не мониторятся); для пульта и ответа нужен **tmux**.
- **UI и голос пока русскоязычные.** Мониторинг, уведомления, диктовка и пульт работают независимо от языка; английская локализация — в планах.
- **Effort** снаружи не читается → панель ведёт оптимистичное состояние.
- **Схемы хуков дрейфуют между версиями агентов.** Если после апдейта `claude` события перестали приходить — сверь с актуальной докой по hooks и поправь `EVENTS`/формат в `src-tauri/src/bin/setup.rs`.
- Жёстко убитый терминал (без `SessionEnd`) оставляет висящую сессию — кнопка «Очистить» в панели убирает done/idle.
- **Wake-word** («Hey Jarvis») — экспериментальный: верификации говорящего нет, модель детектора некоммерческая (см. [«Лицензия»](#лицензия)), по умолчанию выключен. Дев-сборка (`npm start`) включает фича-флаг, но детектор инертен, пока не включишь в панели.
- **Agent-chat** (capability-платформа, MCP-мост `jarvis-mcp`) — мост есть, UI не зашипан. *In progress.*

## Версии

Pre-1.0 (**SemVer 0.x**): контракт хуков и форматы на диске (`~/.jarvis/`) могут меняться. Бамп `0.MINOR` — фичи/breaking, `0.x.PATCH` — фиксы. `1.0` заморозит контракт хуков и формат состояния. Обновления не трогают твои данные: всё пользовательское лежит в `~/.jarvis/`, вне бандла приложения — см. [docs/release/versioning-and-migration.md](docs/release/versioning-and-migration.md).
