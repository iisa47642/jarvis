# Security Policy / Политика безопасности

## Supported versions

The project is **pre-1.0** — the latest released version and the `master` branch are supported.

## How to report a vulnerability (EN)

**Please do not open public issues for security vulnerabilities.**

Report privately via either:

1. **GitHub Security Advisories** (preferred) — the **Security → Report a vulnerability** tab in this repository, which opens a private channel with the maintainer.
2. **Email:** gebodiksch@gmail.com.

Include what you found, reproduction steps, and potential impact. I aim to respond within **72 hours** and keep you updated until the issue is resolved. Credit in the release notes is happily given (if you want it).

## Как сообщить об уязвимости (RU)

**Пожалуйста, не открывай публичные issue по уязвимостям.**

Сообщить приватно можно двумя способами:

1. **GitHub Security Advisories** (предпочтительно) — вкладка **Security → Report a vulnerability** в этом репозитории. Это создаёт приватный канал между тобой и мейнтейнером.
2. **Email:** gebodiksch@gmail.com.

Опиши, что нашёл, как воспроизвести и какое потенциальное влияние. Я постараюсь ответить в течение **72 часов** и держать тебя в курсе, пока проблема не будет закрыта. После выпуска фикса — с удовольствием упомяну тебя в благодарностях (если захочешь).
