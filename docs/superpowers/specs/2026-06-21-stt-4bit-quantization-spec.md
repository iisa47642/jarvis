# STT: 4-bit квантизация qwen3-0.6b — спека (отложено)

> Статус: **отложено по решению пользователя** (2026-06-21). Не реализуем сейчас.
> idle-stop уже снял главную боль (RAM в простое), поэтому квантизация — наименее
> приоритетный рычаг с реальным риском качества ASR. Спека — чтобы вернуться осознанно.

## Что это
Веса модели qwen3-0.6b сейчас в fp16 (2 байта/вес). Квантизация в **4 бита** (0.5 байта/вес,
group-wise: блоки ~32–64 веса со своим scale) уменьшает размер ~в 4 раза.

| | сейчас (fp16) | 4-bit |
|---|---|---|
| Диск (веса) | ~1.5 ГБ | ~0.4–0.5 ГБ |
| RAM при загрузке | ~1.3 ГБ | ~0.4–0.5 ГБ |
| Скорость | базовая | обычно чуть быстрее (меньше bandwidth) |

## Зачем (и почему ценность СНИЗИЛАСЬ)
До idle-stop главной болью были «1.3 ГБ резидентно после диктовки». **idle-stop это уже
решил** (память возвращается через 10 мин). Поэтому ценность квантизации теперь:
- **−1 ГБ диска** (1.5→0.5 ГБ) — основной выигрыш;
- **−0.8 ГБ RAM во время активной диктовки** (в простое уже 0);
- чуть быстрее cold-start/инференс на Apple Silicon.

## Главный риск — качество ASR
«4-bit почти без потерь» — это в основном про **текстовые** LLM. Здесь **ASR** с
аудио-энкодером, чувствительным к точности: огрубление может заметнее бить по **точности
распознавания**, особенно на **RU/EN code-switching** и редких терминах/именах (ровно то,
ради чего STT нужен). Величина потери заранее неизвестна — нужен замер.

## Подход (когда вернёмся)
1. **Квантизовать рядом, не трогая текущую модель**: новый каталог
   `~/.jarvis*/stt-mlx/models/qwen3-0.6b-q4` (или аналог), исходную fp16 оставить как fallback.
   Инструмент: `mlx_lm.convert -q --q-bits 4 --q-group-size 64` (или ручной `mlx.nn.quantize`),
   проверить, что формат совместим с загрузчиком `qwen3-asr-mlx` (может НЕ грузить quantized —
   проверить первым делом, до всего остального).
2. **Не квантовать всё подряд**: эмбеддинги/нормализации/голову обычно оставляют в fp16 —
   реальная экономия чуть меньше теоретической, зато стабильнее качество.
3. **Конфиг-переключатель**: `stt.engine = "qwen3-0.6b-q4"` как ещё один вариант движка
   (рядом с 0.6b/1.7b/whisper), чтобы переключаться и откатываться настройкой, без потери fp16.
4. **Установщик**: квантованная модель как отдельный артефакт (идемпотентно, atomic),
   либо конвертация на месте из уже скачанной fp16 (без повторной сети).

## Приёмочные критерии (обязательно ДО включения по умолчанию)
- **Замер WER до/после** на реальной русской + смешанной RU/EN речи пользователя
  (несколько типичных реплик с терминами/англицизмами). Без слепого включения.
- Память/диск замерены фактически (ожидание ~0.5 ГБ).
- Загрузка и cold-start работают; idle-stop/прогрев совместимы.
- **Откат тривиален**: переключить `stt.engine` обратно на fp16-вариант.
- Если качество просело заметно для носителя — **не включать по умолчанию**, оставить
  опцией «легче/быстрее, чуть хуже».

## Решение
**Отложено.** idle-stop достаточен сейчас; квантизация ASR несёт риск точности, который
оправдан только если (а) 1.5 ГБ диска реально давит, или (б) есть готовность сравнить
качество и откатиться. Делать только с замером WER, не вслепую.
