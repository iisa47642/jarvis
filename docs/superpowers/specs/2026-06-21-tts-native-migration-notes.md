# TTS: заметки про возможный переезд на нативный движок (без Python+torch)

> Статус: **исследование + отложено**. Сейчас сделан только **idle-stop** TTS-сайдкара
> (дёшево, голос Silero сохранён). Полный уход от Python+torch — на будущее. Этот
> документ фиксирует находки, чтобы не исследовать заново.

## Цель
Убрать второй Python-процесс (Silero TTS-сайдкар, `bin/silero-server.py`) и зависимость
от torch: гонять синтез нативно в Rust. Мотив — упрощение установки (нет torch ~ГБ),
быстрее старт, на один супервизируемый процесс меньше.

## Главный блокер (почему это не «просто переписать сайдкар»)
Модель `v4_ru` грузится как **`torch.package`** (`silero.py`: `PackageImporter(...).load_*("tts_models","model")`),
в который зашиты **И нейросеть, И весь `apply_tts`** — нормализация, SSML, `put_yo`,
и главное **`put_accent` — расстановка ударений** (обученная модель/словарь русского stress).

- ONNX-экспорт Silero вытаскивает **только нейросеть** (токены→звук).
- **Ударения в граф ONNX не входят.** Даже официальные ONNX-модели Silero ждут уже
  проставленные ударения (текст с `+` перед ударной гласной).
- Значит в любом ONNX-пути **акцентуация — отдельная задача**, и это качество русской
  речи, которое носитель сразу слышит (неверное ударение = неестественно).

Наш `bin/silero-server.py` уже делает **свою** нормализацию (латиница→кириллица,
числа→слова) — её переписать на Rust легко. Сложность только в ударениях.

## Варианты переезда (когда вернёмся)

### A. Silero v4 через `ort` + своя акцентуация
- Сохраняет **точно текущий голос** (xenia, v4_ru).
- Нужно: (1) вытащить нейросеть из `torch.package` и экспортировать в ONNX
  (`torch.onnx.export` поверх распакованного jit-модуля; нетривиально — пакет скрывает
  forward); (2) реализовать русскую акцентуацию на Rust — открытый словарь ударений
  (напр. данные `russ`/`ruaccent`) + правила для OOV-слов; (3) символьный маппинг как в
  `tts_utils.prepare_text_input` (алфавит модели, см. `models.yml` tokenset).
- **Риск/объём**: высокий. Многодневная реверс-инженерия + риск ошибок ударения.

### B. Piper (нативный ONNX-TTS) — рекомендация, если допустима смена голоса
- Piper СПЕЦИАЛЬНО под ONNX+native; есть русские голоса; фонемизация/ударения через
  **espeak-ng** (C-библиотека, линкуется). Чисто убирает Python+torch.
- Гоняется через тот же **`ort`**, что уже завезён в инкр.10 (wake-word) — инфраструктура
  есть. Нужен espeak-ng (или `piper-phonemize`) — C-зависимость в бандле.
- **Минус**: другой голос/тембр (не Silero xenia). Качество Piper-ru приличное, но иное.
- **Объём**: средний (модель + фонемизатор + ort-обвязка), риск низкий (Piper зрелый).

### C. (сделано) idle-stop TTS-сайдкара
- Не убирает Python, но глушит процесс по простою (10 мин) и поднимает лениво на
  следующей озвучке. Возвращает ~38МБ модели + питон в тихие периоды. Голос Silero как есть.
- Реализация: `voice/sidecar.rs` (active/last_use/in_use/idle_stop) + `voice/mod.rs`
  (worker лениво поднимает + ждёт готовности; tick глушит по простою). Зеркало STT-idle-stop.

## Рекомендация на будущее
Если цель — **именно убрать Python+torch** и допустима смена голоса → **B (Piper)**: это
инженерно-чистый путь, ложится на уже имеющийся `ort`. Если **критично сохранить голос
Silero** → A, но честно закладывать многодневную работу и риск по ударениям. Пока (C)
idle-stop достаточно как дешёвый промежуточный выигрыш.

## Технические якоря (для того, кто возьмётся)
- Текущий сайдкар: `bin/silero-server.py` (apply_tts с `put_accent=True, put_yo=True`,
  SSML `<prosody rate>`, выход WAV 16-bit моно 48к).
- Модель: `~/.cache/torch/hub/snakers4_silero-models_master/src/silero/model/v4_ru.pt`
  (torch.package, 38МБ). Загрузчик: `src/silero/silero.py::silero_tts` (ветка `v4_`).
- Символы/частота: `models.yml` (`tts_models.ru.<speaker>`: tokenset, sample_rate).
- `ort` уже в стеке (фича `wakeword-ort`, инкр.10) — переиспользовать рантайм/паттерн.
- Спикер по умолчанию `xenia`, частоты 8000/24000/48000, дефолт 48000.
