# Хоткей и настройки панели (Raycast-режим)

Дата: 2026-06-11 · Статус: утверждён (подход A)

## Цель

Панель Jarvis вызывается глобальным сочетанием клавиш, появляется по центру
экрана в стиле Raycast и получает страницу настроек внутри самой панели.

## Решение

Настройки — вторая «вьюха» внутри существующего окна панели (шестерёнка в
шапке переключает список ↔ настройки). Хранение — JSON-файл. Хоткей — через
Electron `globalShortcut`.

## Компоненты

### 1. `src/settings.js` — модуль настроек (main-процесс)

- Файл: `~/.jarvis/settings.json`.
- Дефолты: `{ "hotkey": "Command+J", "notifyDone": true, "notifyWaiting": true, "position": "center" }`.
- API: `load()` (битый/отсутствующий файл → дефолты, молча), `save(patch)`
  (merge + запись), значения неизвестных ключей сохраняются как есть.
- Автозапуск при входе НЕ хранится в файле — читается/пишется через
  `app.getLoginItemSettings()` / `app.setLoginItemSettings()`.

### 2. Глобальный хоткей (`src/main.js`)

- На старте: `globalShortcut.register(settings.hotkey, toggleHotkeyPanel)`.
- Смена хоткея: unregister старого → register нового; при неудаче
  (сочетание занято) — откат на старый и ошибка в UI (через ответ IPC).
- Известный трейд-офф: глобальный `⌘J` перехватывает клавишу во всех
  приложениях (Chrome «загрузки», VS Code «панель»). Пользователь в курсе
  и согласен; хоткей меняется в настройках.

### 3. Поведение панели — два режима показа

- **Hotkey-режим (Raycast):** показ С фокусом (`panel.show()`),
  позиция из настроек. `Esc` (keydown в renderer → `panel:hide`) и `blur`
  скрывают панель; фокус возвращается предыдущему приложению (dock скрыт).
  Повторный хоткей при видимой панели — скрыть.
- **Тихий режим (как сейчас):** клик по трею / по уведомлению —
  `showInactive()`, фокус не ворует, blur-hide ОТКЛЮЧЁН (иначе панель
  спрячется мгновенно). Флаг режима хранится в main при показе.
- Позиция `center`: горизонталь — середина, вертикаль — ~⅓ сверху рабочей
  области; дисплей — тот, где сейчас курсор (`screen.getCursorScreenPoint`).
- Позиция `corner`: правый верхний угол, как сейчас (отступ 12px).

### 4. UI настроек (`src/renderer/`)

- Шестерёнка в шапке панели переключает вьюхи список ↔ настройки.
- Содержимое настроек:
  - **Хоткей-рекордер:** поле; клик → состояние «нажми сочетание…»,
    keydown с модификатором фиксирует аккорд, конвертация в Electron
    accelerator (`Command+Shift+J` и т.п.). Esc отменяет запись.
    Ошибка регистрации показывается под полем, значение откатывается.
  - **Тумблер** «уведомлять, когда закончил ответ» (`notifyDone`).
  - **Тумблер** «уведомлять, когда ждёт тебя» (`notifyWaiting`).
  - **Переключатель позиции:** центр / правый верхний угол.
  - **Тумблер** «запускать при входе» (системный login item).
- Стиль — существующий тёмный (те же CSS-переменные), без новых зависимостей.
- Рендер данных — через `textContent`, как в остальном renderer (без innerHTML).

### 5. IPC (preload)

Добавляются: `settings:get` → объект настроек (+ `openAtLogin`),
`settings:set` → `{ ok: true }` либо `{ ok: false, error }` (для хоткея).
Существующие `state:*`, `panel:hide` не меняются.

### 6. Уведомления

`notify()` для события `stop` проверяет `notifyDone`, для `notification`
— `notifyWaiting`. Выключенный тумблер глушит только уведомление; статус
в панели и счётчик в трее обновляются как раньше.

## Поток данных

renderer (настройки) → IPC `settings:set` → `settings.save()` →
side-effects в main (перерегистрация хоткея / login item) → ответ в renderer.

## Ошибки

- Битый `settings.json` → дефолты, без диалогов.
- Хоткей не зарегистрировался → откат + сообщение в UI настроек.
- `globalShortcut` недоступен до `app.whenReady` — регистрируем после ready.

## Вне скоупа

Миграции настроек, несколько хоткеев, vibrancy/анимации появления,
отдельное окно настроек, упаковка приложения.

## Проверка

Тестов в проекте нет — ручная проверка через `npm run demo`:
хоткей открывает/закрывает по центру, Esc/blur закрывают, тихий показ из
трея не крадёт фокус, тумблеры глушат уведомления, смена хоткея работает,
настройки переживают перезапуск.
