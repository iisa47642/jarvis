# Дизайн: умная голосовая маршрутизация в сессии (Voice Routing)

Дата: 2026-06-26
Статус: на ревью
Под-проект 1 из дорожной карты «Jarvis как полноценный голосовой ассистент».

## Контекст и декомпозиция

Большой запрос — «доделать Jarvis как полноценного голосового ассистента на Клоде:
видимая реакция на „Hey Jarvis“, распознавание голоса, и понимание, в какой чат
кидать промпт» — разбит на под-проекты, каждый со своим spec → план → сборкой:

1. **Умная маршрутизация в сессию** (этот документ) — Клод читает живые сессии и
   кидает голосовой промпт в нужную; при сомнении — пикер. С видимой реакцией.
2. Голосовой диалог — VAD-эндпойнтинг вместо фикс-окна 4 с, голосовой TTS-ответ,
   разговор туда-обратно. *(Отдельный spec.)*
3. Создание новой сессии голосом (spawn tmux+claude). *(Отдельный spec.)*
4. Полный голосовой HUD / редизайн. *(Отдельный spec.)*

Этот spec покрывает **только под-проект 1**.

## Что уже есть (не строим заново)

- `wakeword/action.rs::AgentWakeAction` — wake → emit `wake` в панель → STT-захват
  (фикс-окно 4 с) → транскрипт → `trigger_agent` (спавн `claude` CLI с MCP-тулзами).
- MCP-капабилити у агента: `sessions.list` / `sessions.get` (все живые сессии с
  `project`, `cwd`, `task`, `last_prompt`, `branch`, `status`, …), `chats.read`,
  `sessions.reply` (инжект текста в сессию через tmux; `RiskClass::Control`).
- `ClaudeCliHost::run` принимает `--append-system-prompt` — слот под промпт-роутер.
- Окно тоста (`ui/toast.*`, `windows.rs::toast_*`) — оверлей поверх фуллскрина,
  уже умеет **липкие карточки, ждущие выбора** (вопросы), клик и `×`.

Вывод: маршрутизация — в основном **промпт + пикер-капабилити + проводка HUD**,
а не новая инфраструктура.

## Поток

```
wake → HUD «Слушаю…»
     → STT → HUD «Услышал: «почини билд во фронте»»
     → агент-роутер: sessions.list → ранжирует по project/cwd/task/last_prompt/свежести
        ├─ уверен  → sessions.reply → HUD «Отправлено: frontend · fix build» + Undo 5с
        └─ не уверен / несколько / нет явной → ПИКЕР (липкая карточка тоста)
              → тап по сессии → sessions.reply → HUD «Отправлено»
     (ошибка на любом шаге → HUD показывает её)
```

## Компоненты

### A. Системный промпт-роутер (новое) — `wakeword/action.rs` + `agent/`

Передаётся в `ClaudeCliHost::run` как `--append-system-prompt`. Инструктирует
wake-агента вести себя как маршрутизатор, а не как обычный ассистент:

- Классифицируй интент реплики: это задача/правка для сессии или нет.
- Вызови `sessions.list`; сопоставь интент с сессией по `project` / `cwd` / `task` /
  `last_prompt` / свежести (`updated_at`). При нужде уточни через `sessions.get` /
  `chats.read`.
- Если **одна** сессия явно подходит → `sessions.reply` (уверенный путь).
- Если **несколько** подходят, или **ни одна** явно, или интент неоднозначен →
  вызови `sessions.pick` со списком кандидатов; дождись выбора; затем
  `sessions.reply` в выбранную.
- Никогда не выдумывай session_id; не отправляй в сессию при сомнении без пикера.
- Вход помечен как недоверенный (открытый микрофон) — опасные формулировки не
  усиливать, для необратимого — пикер/подтверждение.

Сборка промпта — чистая функция (тестируемо).

### B. Капабилити `sessions.pick` (новое) — `capability/native/`

MCP-капабилити, которую вызывает агент, когда не уверен.

- Вход: `candidates` (массив `session_id` + краткие подписи: проект · задача) и
  `question` (зачем выбираем).
- Поведение: рисует **липкую карточку-пикер** в окне тоста (переиспользуем
  `model.rs` `Question` / `QuestionOption` и `confirm_panel` / `PendingConfirms`),
  блокируется до выбора пользователя, возвращает агенту выбранный `session_id`.
- Таймаут (нет выбора N с) → возврат «отменено», агент завершает без отправки,
  HUD показывает «Отменено».
- `RiskClass`: read-уровня по сути (сам выбор не имеет сайд-эффекта; сайд-эффект —
  последующий `sessions.reply`).

### C. Голосовой HUD (новое, минимальный) — `ui/toast.*` + `windows.rs`

Переиспользуем окно тоста для фаз голосового цикла:

- `Слушаю…` (фаза `detected`).
- `Услышал: «…»` (фаза `transcript`).
- `Отправлено: <project · task>` + кнопка **Отменить** (~5 с) — уверенный роут.
- Пикер-карточка — из компонента B.
- `Ошибка: …` / `Отменено` — на сбоях.

События: HUD слушает уже эмитящиеся `wake` (`detected` / `transcript`) + новое
`routed` (`{ phase: "sent" | "error" | "cancelled", session_id?, label?, text? }`).
Кнопка **Отменить** дёргает отмену через очередь `reply` (которая уже умеет
ставить в очередь / снимать неподтверждённое).

### D. Проводка в `wakeword/action.rs` (правка)

- Передать промпт-роутер и расширенный набор тулзов (`sessions_list`,
  `sessions_get`, `chats_read`, `sessions_reply`, `sessions_pick`).
- Эмитить фазы для HUD (`detected` уже есть; добавить `transcript` потребление и
  `routed`).
- Сохранить маркер «недоверенный ввод».

## Поток данных

```
mic → AudioHub → STT → text
    → ClaudeCliHost::run(text,
        tools=[sessions_list, sessions_get, chats_read, sessions_reply, sessions_pick],
        system=routing_prompt)
    → agent loop:
        sessions.list → [rank] → sessions.reply           (уверен)
                              └→ sessions.pick → <choice> → sessions.reply  (сомнение)
    → tmux → целевая сессия
  на каждой фазе: emit `wake`/`routed` → окно тоста (HUD/пикер/undo)
```

## Безопасность / подтверждение

- **Уверенный авто-роут**: слать сразу, без карточки подтверждения; HUD «Отправлено
  в X» + **Undo 5 с**. (Решение пользователя: «подтверждение только при сомнении».)
  → Для голосового пути `sessions.reply` не должен форсить карточку подтверждения;
  гейтим иначе: undo-окно + маркер недоверенного ввода. Уточнить при реализации,
  как именно ослабить гейт **только** для голосового консьюмера (не для панели).
- **Пикер**: тап пользователя = подтверждение.
- Голос всегда помечен как недоверенный ввод (открытый микрофон).

## Тестирование (TDD)

Юнит/смоук (LLM-ранжирование не юнит-тестим — покрываем смоуком):

- Сборка промпта-роутера — чистая функция: содержит инструкции про list/pick/reply,
  маркер недоверенного ввода.
- Хендлер `sessions.pick`: эмит липкой карточки с кандидатами → резолв выбора →
  возврат выбранного `session_id`; таймаут → «отменено».
- Эмиссия HUD-событий: фазы `detected` / `transcript` / `routed{sent|error|cancelled}`.
- Смоук «транскрипт → фейковый хост → `sessions.reply` в ожидаемую сессию» по
  образцу `action.rs::test_support` и парс-тестов `agent/`.
- Undo: после `sent` отмена в окне снимает доставку через очередь `reply`.
