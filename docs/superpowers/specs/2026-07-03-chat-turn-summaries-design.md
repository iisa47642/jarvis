# Саммари ходов агента в чате сессии

Дата: 2026-07-03. Статус: черновик — ждёт ревью пользователя (вопросы по UX
задавались, ответа не было; принятые допущения помечены «допущение»).

## Проблема

Чат сессии в панели — сырая лента транскрипта: десятки чипов тулов
(`изменил mod.rs`, `выполнил npm test`, …) вперемешку с фрагментами текста
ассистента. Понять «что по сути произошло за ход» невозможно без чтения всего.

Хочется: один ход агента (= сообщение пользователя → всё, что агент сделал до
Stop) отображается как **одно саммари-сообщение**:

- проза «что сделано» (2–4 предложения, по-русски);
- изменённые файлы — кликабельные чипы (открыть/показать в Finder), особенно
  архитектурные/доковые;
- если агент выдал доку/длинный вывод — сжатый, но информативный дайджест;
- команды/тесты — итоговая строка (`cargo test — 281 ok`).

## UX (допущение: «саммари по умолчанию, детали по клику»)

- Реплики пользователя — как сейчас (пузырь + чипы вложений).
- **Завершённый ход агента** — карточка сводки:
  - проза-саммари;
  - чипы файлов: `📄 src/install/mod.rs · ретраи с backoff` — клик открывает
    файл (⌥-клик / контекст — показать в Finder);
  - блок «Дока» (если есть) — свёрнутый дайджест, разворачивается;
  - строка команд/тестов;
  - кнопка «развернуть» — показать сырые реплики и чипы этого хода inline
    (то, что рисуется сейчас).
- **Текущий (живой) ход** — стрим как сейчас (дельты + чипы). По Stop, когда
  сводка готова, ход схлопывается в карточку (сырьё уезжает под «развернуть»).
- **Старые ходы без кэша**: сразу детерминированная карточка (файлы + сжатый
  финальный ответ через `squeeze_reply`), LLM-прозой дозаполняются последние
  N=5 ходов фоном; более старые — по кнопке «Сводка» на карточке.
- Тумблер в шапке чата «Сводка / Лента» (запоминается в настройках,
  по умолчанию «Сводка») — полный откат к сырой ленте одним кликом.

## Рассмотренные подходы

1. **Гибрид: on-Stop + lazy backfill + кэш (выбран).** Живой ход
   суммаризируется на Stop (только если чат этой сессии открыт — экономия
   токенов), история — лениво при открытии чата (последние 5), остальное по
   кнопке. Всегда есть мгновенный детерминированный слой.
2. Полностью on-demand при открытии чата. Проще, но нет живого схлопывания
   хода и каждый первый показ ждёт LLM.
3. Совсем без LLM: только детерминированные карточки (файлы, команды, сжатый
   финальный ответ). Ноль токенов и мгновенно, но нет прозы и дайджеста доки.
   Не отброшен: это фолбэк-слой варианта 1 (нет `claude`/`codex`, таймаут,
   битый JSON → карточка остаётся детерминированной).

## Архитектура

Принцип — **extract-then-abstract**: факты достаёт код (детерминированно и
проверяемо), LLM пишет только прозу поверх фактов. Ссылки на файлы НИКОГДА не
берутся из LLM-выхода напрямую — только сверяются с извлечённым списком.

### Новый модуль `src-tauri/src/turns.rs`

1. **Сегментация ходов**: `Vec<Value>` (entries бэкенда) → `Vec<Turn>`.
   Граница хода — пользовательский текст (`ChatItem{role:user, kind:text}`;
   tool_result-записи типа `user` границей НЕ являются — `to_chat_items` их
   уже отсеивает). Ключ хода: `uuid` user-записи (Claude) / `ts` первой
   user-реплики (Codex).
2. **`TurnFacts` (детерминированно)** из tool_use-записей хода:
   - `files: Vec<FileTouch{path, kind: edited|created}>` — Edit/Write/
     MultiEdit/NotebookEdit (`file_path`), Codex `apply_patch` (хинт из диффа);
   - `commands: Vec<String>` — Bash (`command` + `description`);
   - `subagents`, счётчики тулов, длительность хода;
   - финальный текст ассистента (все text-блоки после последнего юзер-текста —
     логика `full_final_reply`, но в границах хода);
   - головы (~2КБ) записанных `.md`-файлов — вход для дайджеста доки.
3. **Генерация сводки**: промпт (ниже) → `claude_bin::run_service_llm`
   (существующий служебный LLM: Claude haiku → Codex SDK → codex exec,
   настройка «Под капотом»), таймаут 45с, busy-ключ `turnsum` (как `aisum`).
   Парс строгого JSON c ремонтом (паттерн `ai_toast_summary`: срез `{..}`),
   затем **валидация**: `files[].path` ⊆ `TurnFacts.files` (чужие пути
   отбрасываются), длины полей клампятся.
4. **Кэш**: `~/.jarvis/turn-summaries/<session_id>.json`
   `{ v: 1, turns: { <turn_key>: { summary, files:[{path,note}], docs,
   commands, model, at } } }`. Пересборка только отсутствующих; `v` растёт
   при смене схемы/промпта → инвалидация.

### Триггеры

- **Stop-хук** (`daemon.rs`, рядом с `done_summary`): если tail этой сессии
  активен (чат открыт) — суммаризировать последний ход, эмитить событие.
- **`chat_open`**: вернуть ходы с кэшированными сводками; фоном дозаполнить
  последние 5 ходов без сводки (последовательно, не параллельно — не душить
  служебный LLM).
- **IPC `chat_summarize(session_id, turn_key)`** — кнопка «Сводка» на старом
  ходе.

### IPC / события

- `chat_open` → `{ items, turns: [{key, from_ts, to_ts, card|null}] }` —
  сырые items остаются (нужны для «развернуть» и режима «Лента»), поверх —
  разметка ходов.
- Событие `chat:summary` `{ session_id, turn_key, card }` — рендерер
  подменяет/схлопывает карточку.
- **`file_open(session_id, path, reveal: bool)`** — новый IPC: относительный
  путь резолвится от `cwd` сессии, канонизация + проверка существования;
  `open <path>` (дефолтное приложение) или `open -R` (Finder). Открываем
  только обычные файлы; ошибка — тост.

### Рендерер (`renderer.js`)

- `appendChatItems` дополняется группировкой по ходам: пришёл юзер-текст →
  закрыть предыдущую группу хода. Карточка сводки — новый тип сообщения
  `msg turnsum` (DOM без innerHTML, как весь файл).
- Живой ход: рисуется как сейчас; по `chat:summary` с ключом текущего хода —
  обернуть накопленные узлы в `<details>`-контейнер «развернуть», сверху
  вставить карточку.

## Промпт суммаризации (v1, по итогам ресёрча)

Выводы ресёрча (промпт-инжиниринг саммаризации агент-трейсов, июль 2026):

- **Extract-then-abstract** — главный анти-галлюцинационный рычаг: список
  файлов/команд модель НЕ генерирует, а только аннотирует переданный ей
  готовый список (closed world). Подтверждено литературой по verifiable
  summarization и практикой Claude Code compact / aider (`--weak-model`
  для саммари — ровно наш паттерн «малая модель + структурный итог»).
- **JSON без structured-output API**: ранжирование приёмов — «верни только
  JSON» слабо; few-shot с assistant-примером = чистый JSON сильно стабильнее;
  префилл `{` — лучший приём, но **через CLI `claude -p` недоступен**
  (API-фича) → компенсируем few-shot + срез `{..}` + json-ремонт
  (незакрытые скобки/кавычки, снятие ```-заборов).
- **Map-reduce для одного хода — оверкилл** при бюджете ≤ 9–16К символов;
  вместо него детерминированное бюджетирование, «голова+хвост» для длинных
  выводов (середина наименее информативна — lost in the middle).
- **Русский выход при RU/EN-входе**: жёсткий языковой якорь в начале И в
  конце промпта (recency) + few-shot с русским ответом на EN-heavy входе —
  сильнейший рычаг для малых моделей; правило «проза по-русски,
  идентификаторы as-is, не переводить и не транслитерировать»; пост-проверка
  доли кириллицы в `summary` с одним ретраем (эвристика `ru::has_cyrillic`
  уже есть в `ai_toast_summary`).

Вход собирается кодом (бюджеты: FACTS ~1К, сообщение пользователя ~0.5К,
финальный ответ ~4К, сжатая хроника тулов ~1.5К, головы записанных `.md` ~2К;
суммарно ≤ 9К символов; при переполнении режутся в порядке: хроника тулов →
головы доки → финальный ответ «голова+хвост»).

Структура промпта (few-shot вариант — дефолт; минималистичный без примера —
резерв, если латентность/токены станут проблемой):

```text
Ты суммаризируешь один ход кодинг-агента для ленты чата. Отвечай СТРОГО одним
JSON-объектом, без markdown и текста вокруг.
Правила:
- Пиши по-русски. Пути файлов, команды, имена функций/тестов, флаги —
  оставляй как есть на английском, НЕ переводи и НЕ транслитерируй.
- Используй ТОЛЬКО факты из блока FACTS и текста хода. Не выдумывай файлы,
  команды или результаты, которых там нет.
- files: ровно те пути, что даны в FACTS.files (посимвольно); к каждому
  note — одна фраза до 60 символов, что изменилось.
- summary: 2–5 предложений «что сделано и итог».
- docs_digest: если агент выдал доку/отчёт — сжатый пересказ 3–6 пунктов,
  числа/имена/пути дословно; иначе пустая строка.
- commands: итог команд/тестов одной строкой ('cargo test — 281 ok');
  не было — пустая строка.
Схема: {"summary": string, "files": [{"path": string, "note": string}],
        "docs_digest": string, "commands": string}

[FEW-SHOT: пример входа (EN-heavy FACTS+ход) → пример ответа —
 чистый JSON по-русски с англ. путями as-is]

FACTS:
files: {пути + created/edited}
commands: {команда → exit-код/PASS|FAIL}
---
ХОД:
Пользователь: {user_prompt}
{финальный ответ агента}
{сжатая хроника тулов; головы .md-доки, если писались}

Напоминание: ответ — ОДИН JSON-объект, проза по-русски, идентификаторы as-is.
```

Пайплайн вызова: prompt → `run_service_llm` → парс (срез от первой `{` до
последней `}`) → json-ремонт при сбое → валидация (пути ⊆ FACTS, клампы длин,
доля кириллицы в summary; кириллицы нет → один ретрай) → кэш. Любой сбой →
детерминированная карточка.

Анти-галлюцинации, итоговый список: (1) closed-world FACTS + пост-валидация
путей кодом; (2) явный запрет выдумывать + «пустая строка, если нет» для
docs_digest/commands (иначе малые модели заполняют поля выдумкой); (3) числа
и идентификаторы в digest — дословно; (4) фолбэк-слой без LLM всегда есть.

## Ошибки и фолбэки

- Нет ни одного служебного бэкенда → LLM-слой молча выключен, карточки
  детерминированные, кнопок «Сводка» нет.
- Таймаут/битый JSON → детерминированная карточка остаётся, ретрая нет
  (кнопка «Сводка» позволяет повторить вручную).
- Файл из чипа не существует (переименован/удалён) → тост «файл не найден»,
  чип остаётся (история правдива).
- Кэш битый/старая версия → пересборка лениво, файл перезаписывается.

## Тесты

- Сегментация ходов: синтетические entries Claude (uuid-цепочка, tool_result
  не рвёт ход) и Codex (rollout).
- Извлечение фактов: Edit/Write/Bash/apply_patch → пути/команды.
- Бюджеты промпта: длинный ход режется в заданном порядке.
- Парс/ремонт JSON + отбрасывание чужих путей; summary без кириллицы →
  один ретрай, после второго сбоя — детерминированный фолбэк.
- Кэш: чтение/запись, инвалидация по `v`.
- `file_open`: резолв относительного пути от cwd, отказ на несуществующем.

## Вне скоупа

- Саммари для окна `agent-chat` (чат с самим Jarvis) — отдельная лента,
  не транскрипт.
- Ускорение служебного LLM (отдельный трек — docs/codex-summarizer-research-prompt.md).
- Семантическое выделение «архитектурных» файлов — в v1 все файлы равны,
  .md-файлы дополнительно питают docs_digest.
